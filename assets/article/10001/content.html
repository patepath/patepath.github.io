In Angular CLI version 6 or later, we can create multiple applications in a single project or workspace. It is suitable for an enterprise that use a single repository and configuration for all Angular projects.

Advantages

  • We can share library and configuration for all projects
  • We will be got shared a node_modules folder, you don't need to run npm for each project
  • We can handle with a single git repository

Let start

First, letâ€™s start by defining what is an Angular Workspace:

ng new workspace --create-application false

And then, change the directory to workspace folder and generate app1 and app2 applications in this workspace. It will create an application file structure in the workspace/projects folder.

cd workspace
ng g application app1 --routing
ng g application app2 --routing

Run the applicaiton

ng serve --project app1

Or

ng serve --project app2

You can run app1 or app2 but can not run both simultaneously because all applications use the same port. If you want to combine app1 and app2 together. You have to inject sub-appliction (app2) to main-application (app1).
